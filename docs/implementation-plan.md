# 实现计划

## 第一阶段范围

第一阶段先完成业务系统主体，不接钉钉机器人，不做真实销售分配。

核心能力：

- 用户注册、登录、修改资料
- 角色权限：管理员、组长、员工
- 岗位预留：运营、销售
- 运营录入客户线索
- 抖音截图、微信截图上传
- 客户编号自动生成且不可修改
- 客户列表、筛选、搜索、分页、详情
- 成单补录
- Excel 导出
- Word 模板生成与下载
- 基础统计与绩效查看
- 移动端适配，优先保证录入和列表可用

## 开发顺序

1. 完成项目骨架、数据库表结构、基础配置。
2. 完成注册登录、JWT 鉴权、当前用户接口。
3. 完成员工管理、分组管理和数据权限。
4. 完成客户线索新增、编号生成、图片上传。
5. 完成客户列表、详情、筛选、分页。
6. 完成成单补录和状态流转。
7. 完成 Excel 导出。
8. 完成 Word 模板生成。
9. 完成统计页面。
10. 业务测试通过后再接销售分配和钉钉日报/月报。

## 移动端要求

移动端重点服务信息上传场景：

- 底部或顶部导航保持清晰，避免复杂多级菜单。
- 客户录入表单使用单列布局。
- 上传控件支持手机相册选择。
- 列表页在移动端使用紧凑卡片，在桌面端使用表格。
- 筛选条件在移动端收纳为抽屉或折叠区域。
- 主要按钮固定在安全区域内，便于单手操作。
